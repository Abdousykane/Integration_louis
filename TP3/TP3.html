Travaux Pratiques N°3

Objectif : Comprendre les Media Queries pour adapter l'affichage aux écrans mobiles, tablettes et ordinateurs.

Exercice 1 : Changement de couleur au breakpoint

Redimensionnez la fenêtre pour voir le rectangle changer de couleur (Bleu > 960px, Jaune < 960px, Rouge < 480px).

Analyse de l'exercice

Connaissances

J'ai redécouvert la syntaxe de base des @media screen and (max-width: ...). Cela permet d'appliquer des règles CSS conditionnelles basées sur la largeur du viewport, une notion qui m'était légèrement obscur mais que je comprend mieux dès à présent.

Critique

Exercice très simple mais essentiel pour visualiser instantanément l'effet d'un point de rupture (breakpoint).

Difficulté : 1/5.

Pas grand chose à dire, c'était relativement aisé.

Une mise en page complète qui passe de 3 colonnes (Desktop) à 2 colonnes (Tablette) puis 1 colonne (Mobile).

Analyse de l'exercice

Connaissances

J'ai combiné display: grid avec les Media Queries. Au lieu de changer la couleur, j'ai modifié la propriété grid-template-columns pour passer de repeat(3, 1fr) à 1fr selon la taille de l'écran.

Critique

C'est un cas d'usage très concret. Le passage en colonne unique sur mobile améliore grandement la lisibilité du texte. C'est grandement utile pour faire de la mise en page sur mobile.

Difficulté : 2/5.

Lors du début de réalisation de ce TP, j'avais utilisé des "px" comme unité mais çela m'a plus fait perdre du temps qu'autre chose, l'utilisation de "fr" c'est revelé plus pratique.

Exercice 3 : Menu Hamburger

Réduisez la largeur sous 980px : le menu horizontal disparaît et une icône apparaît. Cliquez dessus pour voir le menu.

Nunc eu ullamcorper orci. Quisque eget odio ac lectus vestibulum faucibus eget in metus.

Analyse de l'exercice

Connaissances

J'ai appris à gérer l'affichage conditionnel (display: none/block) des éléments d'interface. J'ai aussi utilisé un peu de JavaScript pour gérer l'événement click sur l'icône, qui ajoute une classe CSS pour afficher le menu mobile.

Critique

Comme pour l'exercice précedant, cette exercice nous permet de mieux gérer les transition entre le mode "Desktop" (menu horizontal) et "Mobile" (menu caché) est fluide. C'est une pattern d'interface standard indispensable aujourd'hui.
De plus c'est le 1er exercice de cette série de TPs qui m'a fait utilisé JavaScript qui est langage très important à connaître pour faire du Front-End.

Difficulté : 3/5.

J'ai eu quelques problème à faire marcher le menu hamburger, car je ne savais pas comment utiliser l'évenement "click" JavaScript pour cette exercice, mais j'ai finalement compris en utilisant "cursor: pointer;" pour mon burger-icon.
